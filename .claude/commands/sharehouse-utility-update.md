# Sharehouse Utility Update

シェアハウスの電気代・ガス代を、Gmail の楽天カード利用通知メールから取得し、Notion の「お金周り」データベースに記入します。

## 対象 Notion データベース

- ページ: 「生活費管理」 https://app.notion.com/p/24e70a52207f81f5970ff165b5f028de
- データソース: 「お金周り」 `collection://24e70a52-207f-814b-994a-000b872531f9`
- 主なプロパティ:
  - `Number`: 金額（支出は**負の値**で記入。例: `-7623`）
  - `date:Transaction Date:start`: 対象月の月末日（例: `2026-06-30`）
  - `支出・収入`: マルチセレクト。電気は `支出-電気`（`支出-通信費` が併記される月もある）、ガスは `支出-ガス`
  - `memo`: 判断根拠を残す（何月分か、引き落とし日、速報版/確定版どちらのメール由来か）

## 記入規則（既存レコードから確定済み）

| 項目 | 契約                                             | 引き落とし                                               | 行の対応規則                                            |
| ---- | ------------------------------------------------ | -------------------------------------------------------- | ------------------------------------------------------- |
| 電気 | 楽天でんき（利用期間: 前月15日〜当月14日）       | 楽天カード（ＪＣＢ）、翌月8日ごろ                        | M月末日の行 = M月分（＝翌月8日の引き落とし額）          |
| ガス | 楽天ガス（利用期間: 前月10日ごろ〜当月13日ごろ） | 楽天カード（ＪＣＢ）、料金確定と同日ごろ（毎月24〜25日） | M月末日の行 = M月中に引き落とされた額（＝前月分の料金） |

## 手順

1. Notion の「お金周り」を SQL でクエリし、`支出-電気` / `支出-ガス` タグ付きで `Number` が NULL の行（未記入月）を洗い出す。
2. Gmail を検索して金額を特定する:
   - 料金確定の通知（金額は載っていない。何月分がいつ確定したかの確認用）:
     `from:energy.rakuten.co.jp`
   - 金額の取得（確定版メールに利用先と金額が載る）:
     `from:rakuten-card.co.jp "楽天でんき"` / `from:rakuten-card.co.jp "楽天ガス"`
   - 確定版が未着の期間は速報版（`【速報版】カード利用のお知らせ`）の JCB カード・利用日・金額から推定できるが、利用先名が載らないため memo に「速報版由来・要確認」と明記する。
3. 上記の記入規則に従って該当月の行の `Number`（負の値）と `memo` を更新する。行がなければ月末日付で新規作成する。
4. まだ請求が来ていない月（電気は翌月8日まで金額未確定）は、memo に「7/8引き落とし予定、金額確定後に記入」等を書いて残す。
5. 更新結果（何月分をいくらで記入したか）をユーザーに報告する。

## 注意

- 電気の行に `支出-通信費` タグが付いている月があるが、金額は楽天でんき単体の引き落とし額が記録されている（過去実績と一致確認済み）。
- 金額が推定（速報版由来）の場合は必ず memo にその旨を残すこと。
